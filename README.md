  "content": "# DriftWatch\n\nDriftWatch is a configuration auditing tool designed to detect and automatically remediate discrepancies between defined infrastructure policies and live environment states. It provides a programmatic way to enforce infrastructure standards and maintain visibility into configuration drift.\n\n## Problem Statement\nIn production environments, manual changes frequently referred to as hotfixes are often applied directly to resources without being reflected in the version controlled configuration. This leads to configuration drift, making infrastructure difficult to replicate, audit, and secure.\n\n## Interactive Demo\nExperience the full lifecycle of drift detection and remediation without manual configuration. The interactive demo simulates real world Chaos Engineering scenarios, including unauthorized changes and policy violations.\n\nTo launch the guided experience:\n\n    npm run demo\n\n## Current Functionality\n* Policy as Code: Infrastructure requirements are defined using standardized YAML syntax.\n* Multi Resource Support: Automatically crawls the policies directory to audit multiple resources simultaneously.\n* Schema Enforcement: Utilizes Zod for strict type validation of ports and protocols.\n* Drift Detection: Real time comparison between Desired vs. Live state.\n* Formatted Reporting: Terminal based dashboard using tables for high visibility.\n* CLI Arguments: Support for --fix flag to toggle between Dry Run and Remediation.\n* Historical Logging: Persistent SQLite database storage for long term compliance tracking.\n* Audit History: Dedicated --history flag to view past audit performance and trends.\n* CSV Export: Automated reporting via --export flag to generate timestamped CSV audit logs.\n* Live Web Dashboard: GitHub Pages integration showing real-time system health.\n* Unit Testing: Automated logic verification via Jest for engineering reliability.\n\n## Technical Stack\n* Language: TypeScript\n* Database: SQLite3\n* Validation Framework: Zod\n* Configuration Format: YAML (Policy) / JSON (Live State)\n* Runtime: Node.js\n* License: MIT\n\n## Getting Started\n\n### 1. Installation\nInstall the necessary dependencies to handle YAML parsing, database operations, and TypeScript execution:\n\n    npm install\n\n### 2. Manual Usage\n\nManual Audit (Dry Run):\n\n    npm start\n\nManual Fix (Remediation):\n\n    npm start -- --fix\n\nView Audit History (Database Logs):\n\n    npm run history\n\nExport Audit History (CSV):\n\n    npm run export\n\nRun Automated Unit Tests:\n\n    npm test\n\n## Project Roadmap\n\n### Core Engine (Completed)\n* ðŸ”µ Phase 1: Policy Validation: Ensure YAML configurations are schema
